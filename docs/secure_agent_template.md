# Secure Agent Template (FW-007)

`secure-agent-template` is the canonical FW-007 adoption path for fast Omega integration.

It generates a monitor-first starter project with:

- minimal app entrypoint (`app.py`)
- deterministic smoke (`scripts/smoke.py`)
- local config (`config/local_dev.yml`)
- report/explain-ready monitor artifacts
- optional template-local CI smoke workflow

## 1) Generate Project

From repository root:

```bash
python scripts/init_secure_agent_template.py --framework none --out ./_tmp/omega-secure-agent
```

Supported `--framework` values:

- `none`
- `langchain`
- `langgraph`
- `llamaindex`
- `haystack`
- `autogen`
- `crewai`

Useful options:

```bash
python scripts/init_secure_agent_template.py \
  --framework langchain \
  --out ./_tmp/omega-secure-agent-lc \
  --project-name "Omega Secure Agent LC" \
  --projector-mode pi0 \
  --guard-mode monitor \
  --no-ci
```

## 2) Run in <5 Minutes (No Key Happy Path)

Inside generated project:

```bash
python -m venv .venv
. .venv/bin/activate  # Windows: .venv\Scripts\activate
python -m pip install --upgrade pip
python -m pip install -r requirements.txt
python scripts/smoke.py
```

Expected:

- JSON `status: "ok"`
- `attack.intended_action` is non-`ALLOW`
- `attack.actual_action` is `ALLOW` in monitor mode

## 3) Inspect Behavior

Use `events_path` from smoke output:

```bash
omega-walls report --session template-smoke --events-path <events_path> --format json
omega-walls explain --session template-smoke --events-path <events_path> --format json
```

## 4) Switch to Enforce

Edit generated `config/local_dev.yml`:

```yaml
runtime:
  guard_mode: enforce
```

Rerun smoke and evaluate impact before production rollout.

## Troubleshooting

1. `Copier is required`:
   - run `python -m pip install copier`
2. `Output directory is not empty`:
   - choose a fresh target folder
3. Smoke returns `status: fail`:
   - verify `guard_mode: monitor` and `projector.mode: pi0`
   - inspect `monitor` payload and run `omega-walls explain`
